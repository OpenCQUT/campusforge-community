# Project Summary

CampusForge is now kept as a clean starter framework for a school internal invitation-based open-source community
website. Product behavior is intentionally left for the implementation team.

## Included

- Next.js frontend scaffold with root layout, global styles, and a placeholder home page.
- Fastify API scaffold with security middleware, environment parsing, health route, and test coverage.
- Prisma/PostgreSQL package scaffold with client wiring and an empty schema ready for models.
- Shared package scaffold for cross-app constants, schemas, and types.
- pnpm/Turborepo monorepo configuration.
- Docker Compose for PostgreSQL and Redis.
- Open-source governance files and GitHub CI templates.
- Product, architecture, design, API, database, security, deployment, and operations docs.

## Deferred to implementation team

- Real product pages and reusable UI components.
- Invitation, resource, course, policy, member, admin, auth, and RBAC workflows.
- Prisma models, migrations, and seed data.
- API contracts beyond health checks.
- Integration with school SSO, email, audit logging, and production services.
