# 01. 产品需求

## 角色

| 角色 | 权限摘要 |
|---|---|
| Visitor | 查看首页、提交邀请申请、使用 GitHub/学校身份入口。 |
| Applicant | 查看自己的申请状态、补充材料。 |
| Member | 查看资源、课程、成员目录、制度文档。 |
| Maintainer | 创建和维护资源、课程、项目入口。 |
| Admin | 审核申请、管理制度、成员、课程、资源和审计日志。 |

## 核心流程：邀请申请

1. 用户输入学校邮箱或学号。
2. 系统检查是否已有邀请或申请。
3. 用户提交申请资料。
4. 系统进入身份校验与管理员审核。
5. 管理员批准、拒绝或要求补充信息。
6. 用户通过邮箱或站内状态页查看结果。

## 资源中心

资源类型：

- Guide
- Template
- Tool
- Policy
- Internal Doc

资源需要支持搜索、分类、收藏、外链、版本更新时间。

## 课程中心

课程类型：

- 自学课程
- 工作坊
- 项目 onboarding
- 活跃 cohort

课程需要支持推荐路径、参与人数、时长、难度标签。

## 制度治理

制度页面展示：

- 行为准则
- 邀请规则
- 资源分享政策
- 安全与隐私
- 课程参与指南

每份制度文档需要状态：`Draft`、`Reviewed`、`Active`、`Archived`。
